# 安全规则

video-review 的核心安全原则是：先 Review，浏览器弹窗确认后，再执行删除。

## 当前版本安全状态

当前版本允许在 Web 页面中删除已标记为“待删除”的媒体文件。

它会：

- 创建 Review 任务记录
- 初始化 SQLite 数据库
- 展示 Web 页面
- 返回 API 数据
- 扫描允许的媒体目录
- 生成截图
- 在用户点击“删除文件”并通过浏览器弹窗确认后，直接删除当前目录下标记为“待删除”的文件

它不会：

- 移动文件
- 重命名文件
- 跳过浏览器确认直接删除文件
- 删除未标记为“待删除”的文件

## 长期安全规则

1. Review 阶段只保存用户决策，不自动修改媒体文件。
2. Web UI 只能删除用户已标记为“待删除”的文件。
3. 点击“删除文件”时必须弹出浏览器确认框。
4. 删除操作直接删除文件，不走 Hermes 审批或 HTTP hook。
5. 执行整理/移动/重命名等非删除操作仍需后续单独设计。
6. 最近修改、下载未完成、疑似被占用或临时文件默认跳过。
7. 每次执行都必须记录 before/after 路径。
8. 单个文件执行失败时记录错误并跳过，不应导致整个任务失控。
9. 不应要求用户修改 NAS 全局 Docker 网络、DNS 等高风险配置作为项目运行前提。
10. Lucky 反代必须开启 HTTPS 和认证，不应裸露到公网。

## 删除策略

当前删除策略是直接删除：

1. 用户先逐个把文件标记为“待删除”。
2. 页面顶部出现“删除文件（N）”按钮。
3. 用户点击按钮后，浏览器显示确认弹窗。
4. 用户确认后，后端调用文件系统删除接口删除这些文件，并从任务数据库移除对应条目。

部署时媒体目录必须以读写方式挂载，否则后端会因为权限不足返回删除失败。
